# MIZAN_CODE_MAP_CONTRACT_AR — عقد ربط Code_Map مع بوابة FINEXSA

**الحالة:** نسخة عربية تنفيذية مؤقتة قبل استيراد Code_Map الرسمي الكامل  
**النطاق:** Client Portal + Excel Import + Account Mapping + MIZAN Legacy Engine  
**الهدف:** منع إدخال أي حساب إلى محرك MIZAN بدون ربط واضح بالترميز المالي الصحيح.

---

## ١. القرار الأساسي

واجهة العميل تكون بالعربي حالياً، لكن المحرك الداخلي يبقى ملتزماً بمنطق MIZAN.

العميل يرى أسماء مفهومة مثل:

- صافي المبيعات
- تكلفة المبيعات
- مصاريف إدارية
- نقدية
- ذمم مدينة
- مخزون
- أصول ثابتة
- موردون
- رأس مال

أما النظام داخلياً فيجب أن يربط كل حساب مع:

- Account Code
- Nature
- Aggregate Code
- Aggregation Sign
- Contribution Rule
- Final Statement Position
- Secondary Aggregate Code عند الحاجة

---

## ٢. ما يظهر للعميل

في قالب Excel الحالي تظهر أعمدة عربية سهلة:

- اسم الحساب
- رقم الحساب
- الرصيد الافتتاحي
- أشهر السنة
- التصنيف المالي
- القسم
- ملاحظات

هذه الأعمدة ليست Code_Map النهائي. هي فقط مدخلات بشرية أولية.

---

## ٣. ما يبقى داخلياً

هذه الحقول لا تظهر للعميل في أول نسخة، لكنها يجب أن تكون موجودة في Mapping لاحقاً:

- mappedCode
- aggregateCode
- contributionRule
- aggregationSign
- nature
- finalStatementPosition

لا يجوز تشغيل MIZAN إذا بقي حساب جوهري بدون Mapping.

---

## ٤. قاعدة التصنيف العربي المؤقت

التصنيفات العربية الحالية في القالب مؤقتة لتسهيل التجربة:

- صافي المبيعات
- تكلفة المبيعات
- مصاريف إدارية
- مصاريف بيعية وتسويقية
- إيرادات أخرى
- نقدية
- ذمم مدينة
- مخزون
- أصول ثابتة
- موردون
- التزامات أخرى
- حقوق ملكية

هذه القائمة ليست بديلاً عن Code_Map الرسمي.  
في Milestone 4 يجب ربط كل تصنيف عربي مع كود MIZAN الرسمي.

---

## ٥. قاعدة الشهرية والرصيد

في النسخة العربية الحالية:

- حسابات قائمة الدخل: القيم الشهرية تمثل حركة الشهر.
- حسابات المركز المالي: القيم الشهرية تمثل رصيد نهاية الشهر.
- المخزون: يحتاج معالجة خاصة حسب سياسة الجرد.
- الرصيد الافتتاحي: يجب أن يكون واضح المصدر قبل تشغيل التقرير.

---

## ٦. قواعد منع التشغيل

يمنع تشغيل MIZAN إذا تحقق أحد الشروط التالية:

- حساب جوهري بدون تصنيف مالي.
- تصنيف عربي غير مربوط بكود MIZAN.
- رصيد افتتاحي مفقود لحساب يحتاج افتتاحي.
- شهر مالي غير قابل للقراءة.
- ملف لا يحتوي الأعمدة المطلوبة.
- سياسة المخزون غير محددة عند وجود حسابات مخزون.

---

## ٧. المطلوب قبل Milestone 4

قبل شاشة Mapping النهائية يجب توفير Code_Map الرسمي أو تصديره إلى ملف يحتوي الأعمدة التالية:

- Account Code
- Nature
- Closing Statement
- Arabic FS Classification
- English Classification Group
- Parent Account
- Final Statement Position
- Aggregate Code
- Aggregation Sign
- Secondary Aggregate Code
- Contribution Rule
- Debit/Credit Movement Sign

بعد توفره يتم إنشاء جدول ربط عربي:

التصنيف العربي → Code_Map الرسمي → Aggregate Code → Rule

---

## ٨. قاعدة عدم كسر الترميز

لا يجوز تغيير رموز MIZAN القديمة.

أي تطوير في الموقع يجب أن يحافظ على:

- Account Code
- Aggregate Code
- الرموز التي تنتهي بـ ? أو !
- قواعد المخزون
- قواعد Opening Balance
- قواعد Target
- قواعد Department

الواجهة العربية هي طبقة عرض فقط، وليست بديلاً عن العقل المالي.
